## Why this matters

Four direct dependencies in `package.json` are not imported anywhere in
`src/`. They bloat `node_modules` and the lockfile for no benefit. Two of
them (`lowlight`, `highlight.js`) are already pulled in transitively by
`rehype-highlight`, so listing them directly is redundant. Removing them
shrinks the install and removes confusion about which highlighter the project
actually uses (answer: `rehype-highlight` for MDX fences, `prism-react-renderer`
for the `<CodeBlock>` component).

## Steps

### Step 1: Re-confirm the packages are unused

```bash
rg -n "react-syntax-highlighter|lowlight|highlight\.js" src/
```

**Verify**: no matches in `src/` (the `highlight-js` CSS path under
`src/styles/` matches the regex `highlight\.js`? No — it is `highlight-js`
with a hyphen, and the regex `highlight\.js` requires a literal dot, so it
will NOT match `highlight-js`. Confirm the only matches, if any, are the CSS
import comment, not a JS import of the package.)

If you see an actual `import ... from "highlight.js"` or
`from "react-syntax-highlighter"` or `from "lowlight"`, STOP — it is in use;
report back.

### Step 2: Remove the four entries from package.json

Expected: all three directories exist (`lowlight` and `highlight.js` remain as
