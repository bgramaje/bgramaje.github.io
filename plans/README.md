## Findings considered and rejected

- **`Date.now()` as React key in `HomePage`/`TerminalModal` history items
  (`src/pages/HomePage.tsx:24,96`, `src/components/terminal/TerminalModal.tsx:53,85`)**:
  potential key collision if two commands run in the same millisecond. Not
  worth a plan — collisions are invisible in practice (React dedupes by key,
  worst case is a dropped warning) and the fix (a monotonic counter) touches
  hot path state for no user-visible gain. Revisit only if history bugs appear.
- **`BitcoinTicker` swallows fetch errors silently
  (`src/components/BitcoinTicker.tsx:75`)**: the `.catch(() => {})` is correct
  for a non-critical decorative ticker; surfacing errors would add noise.
  Not worth doing.
- **`MermaidDiagram` separate from `mdx-mermaid`**: resolved by deletion in
  Plan 002 (it is dead code), not by reconciling the two implementations.
- **Adding a test framework + `processCommand` unit test**: flagged as a
  direction option but not selected this round. The pure `processCommand`
  function is the one piece where a ~40-line vitest test would pay for
  itself; deferred until the maintainer wants a test baseline.
- **Tailwind v3 → v4 / React 18 → 19 migration**: explicitly out of scope per
  AGENTS.md ("Do not migrate to Tailwind v4 or React 19 without an explicit
  request"). Not audited as a finding.
- **Blog locale routing via `?lang=` query param instead of a URL path
  (`/blog/:id/:locale`)**: flagged as a direction option in the second audit.
  Path-based locales index better in search engines and make `hreflang`
  viable; the current query-param approach works and is bookmarkable. Not
  promoted to a plan — it is a route refactor with redirects and is only
  worth it if SEO indexing of both language variants becomes a goal.
- **`<html lang="en">` while content is bilingual/Spanish** (`index.html:2`):
  a global i18n nit, not blog-specific. Left for a future global-i18n pass;
  not planned here.
- **`BlogListPage` header "subtitle" regression**: investigated and rejected —
  the remote version (`abad841:BlogListPage.tsx`) already had only
  `<h1>Blog</h1>` before the merge; no regression occurred.
- **`BlogPost` `?lang` sync `useEffect` infinite-loop risk**: reviewed; the
  guards (`if (!langParam || !locales.includes(langParam))`) prevent a
  re-set once the param matches the effective locale. By-design, not a bug.
- **CV PDF chunk size (1.57 MB)**: `CvPdfDownloadButton` lazy-`import()`s the
  PDF renderer on click, so it is correctly code-split and excluded from the
  initial bundle. Not a finding.
